# SRS: Интернет-магазин автомобильных запчастей «Spare Parts for Everyone»

## 1. Введение

### 1.1 Цель документа

Целью данного документа является формализация требований к веб-платформе «Spare Parts for Everyone», предназначенной для взаимодействия между покупателями и продавцом в процессе поиска, выбора и приобретения автомобильных запчастей онлайн.  
Документ служит основой для согласования ожиданий между заказчиком, командой разработки и тестирования.

### 1.2 Область действия

В рамках данного проекта реализуется веб-платформа интернет-магазина автомобильных запчастей, включающая следующие ключевые компоненты:

- Пользовательская часть:
  - Поиск запчастей **по VIN-коду автомобиля**
  - Поиск запчастей **по номеру (артикулу) производителя**
  - Каталог с фильтрацией по марке, модели и году выпуска
  - Корзина и оформление заказа с онлайн-оплатой
  - Личный кабинет покупателя (история заказов, личные данные)

- Административная часть:
  - Управление товарами, категориями и справочниками (автомобили, производители)

**Не входят в текущую версию (MVP):**
- Мобильное приложение (iOS/Android)
- Интеграция с внешними ERP/1С-системами
- Система персонализированных рекомендаций
- Сравнение запчастей или "похожие товары"
- Онлайн-чат с поддержкой

### 1.3 Определения, сокращения и термины

| Термин / Сокращение | Определение |
|---------------------|-------------|
| **VIN** | Vehicle Identification Number — уникальный 17-символьный идентификационный номер автомобиля, по которому можно точно определить его марку, модель, год выпуска, двигатель и другие характеристики. |
| **Артикул (OEM-номер)** | Оригинальный номер запасной части, присвоенный производителем автомобиля или поставщиком. Используется для однозначной идентификации товара. |
| **Каталог** | Иерархическая структура товаров, организованная по категориям (например, «Тормозная система»), маркам и моделям автомобилей. |
| **MVP** | Minimum Viable Product — минимально жизнеспособная версия продукта, содержащая только ключевые функции, необходимые для запуска. |
| **Покупатель** | Конечный пользователь системы, осуществляющий поиск и заказ запчастей (физическое или юридическое лицо). |
| **Администратор** | Пользователь с правами управления товарами, заказами и справочниками в административной части системы. |

### 2.1 Заинтересованные стороны

| Стейкхолдер                     | Обязанности                                      | Проблемы / Потребности                                                                 |
|----------------------------------|--------------------------------------------------|----------------------------------------------------------------------------------------|
| **Покупатель**                  | Поиск, выбор и заказ запчастей                   | — Быстро найти нужную запчасть по VIN или артикулу<br>— Уверенность в совместимости<br>— Прозрачные цены и условия доставки |
| **Администратор**               | Управление товарами, заказами, справочниками     | — Удобный интерфейс для загрузки и редактирования товаров<br>— Минимизация ошибок при сопоставлении запчастей с автомобилями |
| **Владельцы бизнеса / Инвесторы** | Определение стратегии, утверждение бюджета и приоритетов | — Рост конверсии и выручки<br>— Быстрый запуск MVP<br>— Контроль над ключевыми метриками (ассортимент, остатки, заказы) |
| **Команда продукта**            | Разработка, тестирование и поддержка системы     | — Чёткие, недвусмысленные и тестируемые требования<br>— Стабильный scope на этапе реализации |
