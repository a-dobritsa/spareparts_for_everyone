# SRS: Интернет-магазин автомобильных запчастей «Spare Parts for Everyone»

## 1. Введение

### 1.1 Цель документа

Целью данного документа является формализация требований к веб-платформе «Spare Parts for Everyone», предназначенной для взаимодействия между покупателями и продавцом в процессе поиска, выбора и приобретения автомобильных запчастей онлайн.  
Документ служит основой для согласования ожиданий между заказчиком, командой разработки и тестирования.

### 1.2 Область действия

В рамках данного проекта реализуется веб-платформа интернет-магазина автомобильных запчастей, включающая следующие ключевые компоненты:

- Пользовательская часть:
  - Поиск запчастей **по VIN-коду автомобиля**
  - Поиск запчастей **по номеру (артикулу) производителя**
  - Каталог с фильтрацией по марке, модели и году выпуска
  - Корзина и оформление заказа с онлайн-оплатой
  - Личный кабинет покупателя (история заказов, личные данные)

- Административная часть:
  - Управление товарами, категориями и справочниками (автомобили, производители)

**Не входят в текущую версию (MVP):**
- Мобильное приложение (iOS/Android)
- Интеграция с внешними ERP/1С-системами
- Система персонализированных рекомендаций
- Сравнение запчастей или "похожие товары"
- Онлайн-чат с поддержкой

### 1.3 Определения, сокращения и термины

| Термин / Сокращение | Определение |
|---------------------|-------------|
| **VIN** | Vehicle Identification Number — уникальный 17-символьный идентификационный номер автомобиля, по которому можно точно определить его марку, модель, год выпуска, двигатель и другие характеристики. |
| **Артикул (OEM-номер)** | Оригинальный номер запасной части, присвоенный производителем автомобиля или поставщиком. Используется для однозначной идентификации товара. |
| **Каталог** | Иерархическая структура товаров, организованная по категориям (например, «Тормозная система»), маркам и моделям автомобилей. |
| **MVP** | Minimum Viable Product — минимально жизнеспособная версия продукта, содержащая только ключевые функции, необходимые для запуска. |
| **Покупатель** | Конечный пользователь системы, осуществляющий поиск и заказ запчастей (физическое или юридическое лицо). |
| **Администратор** | Пользователь с правами управления товарами, заказами и справочниками в административной части системы. |

## 2. Общая информация о системе

### 2.1 Заинтересованные стороны

| Стейкхолдер                     | Обязанности                                      | Проблемы / Потребности                                                                 |
|----------------------------------|--------------------------------------------------|----------------------------------------------------------------------------------------|
| **Покупатель**                  | Поиск, выбор и заказ запчастей                   | — Быстро найти нужную запчасть по VIN или артикулу<br>— Уверенность в совместимости<br>— Прозрачные цены и условия доставки |
| **Администратор**               | Управление товарами, заказами, справочниками     | — Удобный интерфейс для загрузки и редактирования товаров<br>— Минимизация ошибок при сопоставлении запчастей с автомобилями |
| **Владельцы бизнеса / Инвесторы** | Определение стратегии, утверждение бюджета и приоритетов | — Рост конверсии и выручки<br>— Быстрый запуск MVP<br>— Контроль над ключевыми метриками (ассортимент, остатки, заказы) |
| **Команда продукта**            | Разработка, тестирование и поддержка системы     | — Чёткие, недвусмысленные и тестируемые требования<br>— Стабильный scope на этапе реализации |

### 2.2 Ролевая модель

| Роль | Описание | Права |
|------|--------|------|
| **Гость** | Пользователь, не вошедший в систему | — Просмотр каталога<br>— Поиск по VIN и артикулу<br>— Добавление товаров в корзину (анонимная) |
| **Покупатель** | Авторизованный пользователь (физическое или юридическое лицо) | — Всё, что доступно гостю<br>— Оформление заказа<br>— Доступ к личному кабинету<br>— Просмотр истории заказов<br>— Редактирование профиля и адресов доставки |
| **Администратор** | Сотрудник магазина, управляющий контентом и заказами | — Управление товарами (создание, редактирование, удаление)<br>— Просмотр и изменение статусов заказов<br>— Управление справочниками (автомобили, категории, производители)<br>— Просмотр списка пользователей |

### 2.3 Ограничения и допущения

#### Допущения
- Покупатель владеет корректным VIN-кодом своего автомобиля или знает артикул нужной запчасти.
- Все автомобили, поддерживаемые системой, имеют стандартный 17-символьный VIN.
- Администратор обладает актуальной и структурированной базой данных запчастей с привязкой к моделям автомобилей.
- Не требуется поддержка многоязычности: интерфейс реализуется только на русском языке.

#### Ограничения
- Система не поддерживает оффлайн-режим: требуется постоянное подключение к интернету.
- Не предусмотрена интеграция с внешними платёжными системами помимо базового онлайн-эквайринга.
- Доступ к административной части возможен только по защищённому HTTPS-соединению.
- Система не хранит полные банковские реквизиты пользователей.

## 3. Требования к системе

### 3.1 Функциональные требования

| ID | Требование | Описание |
|----|-----------|---------|
| FR-01 | Поиск по VIN | Система принимает 17-символьный VIN и отображает список запчастей, совместимых с указанным автомобилем. |
| FR-02 | Поиск по артикулу | Система находит товар по точному совпадению OEM-номера и отображает карточку запчасти. |
| FR-03 | Фильтрация каталога | Пользователь может фильтровать товары по марке, модели, году выпуска и категории. |
| FR-04 | Отображение карточки товара | Для каждой запчасти отображаются: артикул, производитель, изображение, цена, наличие на складе. |
| FR-05 | Уведомление о поступлении | При отсутствии товара пользователь может оставить email для уведомления о поступлении. |
| FR-06 | Анонимная корзина | Пользователь может добавлять и удалять товары из корзины без авторизации. |
| FR-07 | Оформление заказа | При переходе к оформлению неавторизованный пользователь перенаправляется на регистрацию/вход. |
| FR-08 | Данные заказа | Пользователь указывает: способ и адрес доставки, способ оплаты (онлайн-карта). |
| FR-09 | Подтверждение заказа | После оформления система отправляет email с деталями заказа. |
| FR-10 | История заказов | Авторизованный пользователь может просматривать список своих заказов и их статусы. |
| FR-11 | Управление профилем | Пользователь может редактировать личные данные и сохранённые адреса доставки. |
| FR-12 | Управление товарами | Администратор может создавать, редактировать и деактивировать запчасти в каталоге. |
| FR-13 | Управление справочниками | Администратор может управлять списками марок, моделей, лет выпуска и категорий. |
| FR-14 | Управление заказами | Администратор может изменять статус заказа («в обработке», «отправлен», «доставлен»). |

### 3.2 Нефункциональные требования

| Атрибут | Требование |
|--------|-----------|
| **Производительность** | Время загрузки страницы каталога или карточки товара — не более 2 секунд при скорости интернета от 10 Мбит/с. |
| **Доступность** | Система доступна 99% времени в течение месяца (исключая плановые технические работы). |
| **Совместимость** | Веб-интерфейс корректно отображается в последних версиях браузеров: Chrome, Firefox, Safari, Edge (не старше 2 лет). |
| **Адаптивность** | Интерфейс корректно отображается на экранах шириной от 320px (мобильные устройства) до 1920px (десктопы). |
| **Безопасность** | Все формы авторизации и оплаты передаются по HTTPS. Пароли хранятся в хэшированном виде. |
| **Удобство использования** | Пользователь может найти запчасть по VIN или артикулу не более чем за 3 клика. |
| **Масштабируемость** | Система поддерживает до 10 000 товаров и 1 000 одновременных пользователей без деградации производительности. |
| **Поддержка** | Логирование ошибок и ключевых действий (оформление заказа, вход) обеспечивается для диагностики и анализа. |
